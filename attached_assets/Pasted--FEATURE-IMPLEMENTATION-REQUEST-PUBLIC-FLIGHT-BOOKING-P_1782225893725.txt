# FEATURE IMPLEMENTATION REQUEST: PUBLIC FLIGHT BOOKING PORTAL + APPROVAL WORKFLOW

## IMPORTANT ARCHITECTURAL RULES

Before making any changes, follow these constraints:

* DO NOT modify existing GDS integrations.
* DO NOT modify ticket issuance logic.
* DO NOT modify accounting ledgers.
* DO NOT modify supplier APIs.
* DO NOT modify existing ERP booking workflows.
* Reuse existing UI components wherever possible.
* Preserve current database relationships.
* Build this as an isolated extension layer.

The goal is to expose flight search publicly while routing all bookings into a manual approval workflow inside the ERP.

# PHASE 2 — BOOKING REQUEST PIPELINE

## Objective

Prevent automatic ticket issuance from the public website.

### Existing Behavior

Current flow:

Search → Select Flight → Book → Issue Ticket

### New Public Flow

Search → Select Flight → Submit Request

No ticket issuance.

No PNR creation.

No payment capture.

No accounting entries.

No supplier booking.
